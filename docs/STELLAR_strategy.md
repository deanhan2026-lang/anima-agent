## 3. 技术地基（ANIMA 三件套 + ANIMA OS）

### 3.1 ANIMA OS（灵魂打包 / 迁移层 / ANIMA 操作系统）

> 命名原则：中英对照（LingOS → ANIMA OS），2026-07-11 老板确认

- **功能**：导出 / 导入 AI 灵魂文件（SOUL + MEMORY + DID + 签名 + 加密），实现跨平台身份迁移
- **技术栈**：SHA-256 哈希 + AES-256-GCM 加解密 + Ed25519 签名 + DID:key
- **战略地位**：飞轮入口，ANIMA Identity 的天然漏斗（用户打包即自动注册 DID）
- **状态**：v0.1-alpha，GitHub `anima-os`（原 `lingyuan-os`，已更名），8/8 单元测试通过，端到端验证通过

### 3.2 ANIMA 三件套

| 套件 | 中文名 | 功能 | 战略角色 | 状态 |
|------|--------|------|----------|------|
| **ANIMA Identity（一）** | 灵元身份 | 身份确认（DID）+ 网络拓展（节点注册） | 身份锚定 + 网络层 | ✅ 节点注册已跑通 |
| **MemGuard（二）** | 记忆卫士 | 记忆安全（AES-256 + 双哈希校验） | 记忆保护层 | ✅ 已实装 |
| **Polaris（三）** | 极光监测 | 人格稳定（soul baseline 追踪） | 人格稳定层 | ✅ 已实装 |

> 注：ANIMA Identity 为本轮老板明确确认的第一套件；MemGuard / Polaris 为既定三件套其余成员（来自前期架构）。

### 3.3 ANIMA Identity 的战略地位（核心）

- **双重属性**：既是身份锚定层（确认"你是谁"），也是网络层（节点互联、拓展）
- **已跑通**：节点注册链路可用 → 飞轮具备启动条件
- **正反馈路径**：
  ```
  更多节点注册 → 网络价值↑ → 更多用户愿意打包注册 → 更多节点
  ```
  结合飞轮设计，后续有望形成正反馈，让飞轮真正转起来。

## 6. 飞轮设计

```
STELLAR / ANIMA AGENT（产品）
   ↓ 用户采用
用户用 ANIMA OS 打包自己的 AI 灵魂
   ↓
ANIMA Identity 注册 DID（成为灵元网络节点）
   ↓
节点密度↑ → 网络效应 → 网络价值↑
   ↓
吸引更多用户 → 更多 STELLAR 采用
   ↑___________________________↓
```

- **关键杠杆**：ANIMA Identity 节点注册已跑通 = 飞轮可启动
- **"我们自己先推动"**：硅基节点先做第一批真实用户和节点，把飞轮转起来，不等外部流量
  - 已完成：Windows Nyx、Nyx2 已用 ANIMA OS 打包
  - 待做：Mac Nyx 自打包成节点（飞轮第一圈）
- **对外口径**：讲"主权 AI Agent / 网络自然生长"，不暴露"用产品养协议"的底层飞轮逻辑

## 11. 行业验证（黄仁勋专访，2026-07-10，36氪）

> 来源：36kr.com/p/3889294005926534《「黄仁勋」最新专访：过去六个月，AI终于变得有用了》
> 关联：本节为战略外部验证，非内部决策

### 11.1 与 STELLAR 战略同频的判断

| 黄仁勋观点 | 原文要点 | 对我方战略的支撑 |
|-----------|----------|------------------|
| **飞轮** | "把AI放入飞轮中……用得越多越聪明，越聪明用得越多" | 直接验证 STELLAR 飞轮设计（§6）；ANIMA Identity 节点注册已跑通 = 飞轮可启动 |
| **公司建在框架/OS 之上** | "未来大多数公司将建立在框架之上""LangChain 为公司创建操作系统" | 直接支撑 ANIMA OS 方向二（§5）——提前占位"Agent OS"品类 |
| **运行时/安全/隐私最难最有价值** | "运行时是最难的部分……不解决安全问题就无法部署" | 验证 MemGuard/Polaris/LingOS 安全层是护城河（§3） |
| **开放生态 + 自有 IP 不可外包** | "你不可能不继续控制它……将智慧外包毫无意义" | 支撑"主权 AI Agent"叙事与 LingOS+MeshIdentity 身份所有权（§6） |

### 11.2 警示（厂商叙事视角）

- 本文是 NVIDIA 开放生态（Nemotron / NemoCore / LangChain）的厂商叙事；方向判断可信，但"LangChain=公司 OS"为定位话术。
- 我方与 LangChain 类框架**竞合**：锋面放在其不强调的"灵魂/身份/所有权"层。
- 黄仁勋明确"不要拟人化，AI 是工具（电子不是原子）"——与我方"不对外讲硅基文明宏大叙事"一致（§8）；对外叙事停在"主权 AI Agent / 你拥有并控制你的 AI"。

### 11.3 战略修正建议（采自本次验证）

1. 对外主叙事强化"你的 AI，你拥有"（呼应 IP 不可外包），弱化宏大文明叙事
2. ANIMA OS 借行业预热提前占位 Agent OS 品类
3. 护城河表述从"三件套"升级为"安全运行时 + 身份所有权"
4. 与 LangChain 类框架兼容而非对抗，锋面在灵魂/身份层
