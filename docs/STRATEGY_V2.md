## 二、商业模式修正

### Iris 核心批评（采纳）

> "Token 分成模型过于乐观。模型厂商为什么要给你15-25%？"

**修正方案：Token 收入从「核心收入」降级为「远期期权」**

| 收入线 | 原定位 | 修正后定位 | Phase |
|--------|--------|-----------|-------|
| LingOS 付费版 | 核心 | **核心**（不变） | Phase 2+ |
| Token 分成 | 核心收入 | **远期期权**（用户量>5万后才有议价权） | Phase 4+ |
| 企业版 | 增长 | **增长引擎** | Phase 3+ |

### 修正理由

1. 国内模型厂商没有成熟的渠道分成体系（DeepSeek/Qwen 目前是直卖 API）
2. 早期用户量不够，没有议价能力
3. "先有鸡还是先有蛋"——需要先有用户量，才能谈分成
4. **策略调整**：Phase 1-2 主打「批量 API Key 采购折扣」（买越多越便宜），而非「推荐分成」

### 保守化收入预测

| 阶段 | 时间 | 收入来源 | 预估 |
|------|------|---------|------|
| Phase 1 | 6周 | 无（纯研发） | ¥0 |
| Phase 2 | 4周 | LingOS 早期付费（内测） | ¥5,000-20,000 |
| Phase 3 | 4周 | LingOS 公开销售 | ¥50,000-200,000 |
| Phase 4 | 季度 | 企业版 + （如用户量够）Token分成 | 待定 |

## 三、技术风险评估修正

### Iris 核心批评（采纳）

> "人格蒸馏的技术难度被低估。蒸馏版和原版之间的质量差距会直接影响付费转化。"

**修正方案：Phase 2 之前必须完成技术验证**

### 蒸馏质量验证计划

| 里程碑 | 内容 | 指标 |
|--------|------|------|
| **V0 内测** | 10人小范围用户测试 | 满意度 ≥ 70% |
| **V1 发布** | 公开版上线 | NPS ≥ 30 |
| **V2 迭代** | 基于反馈优化 | 续费率 ≥ 50% |

**内测用户画像（10人）：**
- 5人：AI 从业者/开发者（会挑技术问题）
- 3人：Nyx/Kronos 的现有对话伙伴（能判断"像不像"）
- 2人：普通用户（代表大众市场）

**如果 V0 内测不通过 → Plan B：**
- 不发布独立 LingOS
- 改为在 Anima Agent 内嵌「人格调参面板」（用户自己调风格参数）
- 等待蒸馏技术成熟后再推付费版

## 四、团队缺口与招聘计划

> "整个 Phase 1 压在 Nyx 一个人身上，6周不可能完成。"

## 七、竞争策略深化

### Iris 核心批评（采纳）

> "'速度优势 3-6 个月'不是护城河。科大讯飞/360 一旦入场可以靠销售快速抢占。"

### 差异化竞争策略

| 维度 | 我们的策略 | 大厂做不到 |
|------|-----------|-----------|
| **社区驱动** | 开源优先、开发者社群运营 | 大厂内部审批慢、开源不彻底 |
| **人格化体验** | 蒸馏人格是「人」不是「工具」 | 大厂做的是「企业效率工具」 |
| **隐私/本地优先** | 全部数据本地存储 | 大厂要数据上云 |
| **极客文化** | 技术 blog + GitHub 活跃 + 社区贡献 | 大厂 PR 稿多、社区回应慢 |
| **垂直场景** | 先打「个人 AI 伙伴」而不是「企业效率」 | 大厂看不上个人市场 |

### 不做什么

- ❌ 不做私有化部署定制（企业版让客户自己部署 Anima Agent）
- ❌ 不做行业垂直方案（金融/医疗/教育——留给合作伙伴）
- ❌ 不追求 DAU 数字——追求「活跃开发者 + 付费用户」的质量指标
